SPECIFICATION SUMMARY (from input_ui_spec.txt)
────────────────────────────────────

Project: GreenEarth – Environmental Protection Website
Goal: Public-facing site that educates users, promotes sustainable actions, and encourages community participation.
Target Users: General public, students, environmental volunteers, NGOs, activists.

PAGES REQUIRED:
1. Home Page - Hero section with headline, description, "Take Action" CTA button; Featured sections for Climate Change, Wildlife Protection, Sustainable Living; Navigation bar; Footer with copyright and social links.
2. About Page - Mission statement, organization values, environmental goals (static content).
3. Initiatives Page - List of initiatives with title, description, category (Climate/Wildlife/Ocean/Recycling); Category filter; Responsive grid layout.
4. Articles/Knowledge Page - List of educational articles with title, summary, publish date; Clicking opens detail view page; Mock data content.
5. Get Involved Page - Volunteer sign-up form with Name, Email, Area of Interest dropdown, Submit button; Form validation (required fields, email format); Success message after submission (no backend).
6. Contact Page - Contact form with Name, Email, Message; Basic validation; Confirmation message on submit.

TECHNICAL REQUIREMENTS:
- Responsive design (mobile, tablet, desktop)
- Accessible forms and buttons with semantic HTML and ARIA labels
- Tailwind-based styling with eco-friendly color palette
- Mock data stored locally (no real API)
- React Router for navigation
- All data structures must be typed with TypeScript
